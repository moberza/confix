ac_define_dir_

Obsolete Macro

Use AC_DEFINE_DIR instead.

Synopsis

AC_DEFINE_DIR_(VARNAME, DIR [, DESCRIPTION])

Description

This macro _AC_DEFINEs VARNAME to the expansion of the DIR variable, taking care of fixing up ${prefix} and such.

Note that the 3 argument form is only supported with autoconf 2.13 and later (i.e. only where _AC_DEFINE supports 3 arguments).

Examples: 

   AC_DEFINE_DIR_(DATADIR, datadir)
   AC_DEFINE_DIR_(PROG_PATH, bindir, [Location of installed binaries])

This macro is based on Alexandre Oliva's AC_DEFINE_DIR.

Author

Guido Draheim <guidod@gmx.de>

Last Modified

2005-08-01

M4 Source Code 

AC_DEFUN([AC_DEFINE_DIR_], [
  test "x$prefix" = xNONE && prefix="$ac_default_prefix"
  test "x$exec_prefix" = xNONE && exec_prefix='${prefix}'
  ac_define_dir=`eval "echo [$]$2"`
  ac_define_dir=`eval "echo [$]ac_define_dir"`
  ifelse($3, ,dnl
    AC_DEFINE_UNQUOTED($1, "$ac_define_dir"),dnl
    AC_DEFINE_UNQUOTED($1, "$ac_define_dir", $3))
])
